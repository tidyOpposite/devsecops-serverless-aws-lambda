# v1.0.0 Release Candidate Checklist

Target stable release: `v1.0.0`.
Hardening release stream: `v0.12.0`.
Status: `v0.12.0` release-candidate hardening record, not a stable release tag.

Use this checklist before publishing any final `v1.0.0` candidate. The goal is
to make every Version 1.0 criterion traceable to commands, docs, tests,
workflow artifacts, or a release evidence bundle.

Do not mark `v1.0.0` stable while the blocker register in this document has
open items. `v0.12.0` hardening can ship as a pre-stable release while these
stable-release blockers remain explicit.

## Blocker Register For v1.0.0 Stable

These are not blockers for continuing staged `v0.12.0` hardening, but they
must be closed before a stable `v1.0.0` release is tagged.

| ID | Blocker | Required closure evidence |
| --- | --- | --- |
| RC-B1 | A full production deployment walkthrough has not been executed and attached from a real AWS account and GitHub repository. | Completed [Production deployment evidence](production-deployment-evidence.md) bundle with install, setup, workflow, Terraform, AWS, health, CloudWatch, image, rollback-readiness, and lessons-learned artifacts. |
| RC-B2 | GitHub CLI and AWS CLI compatibility were not verified on the current machine because `gh` and `aws` were not installed. | `gh --version`, `aws --version`, `devsecops doctor github --strict --format json`, `devsecops doctor aws --environment prod --strict --format json`, and `devsecops github status --strict --format json` from the target repository/account. |
| RC-B3 | WSL2 Ubuntu support is documented but does not yet have an attached release-candidate transcript. | WSL2 install, `devsecops --version`, first-success dry-run, completion generation, and unit-test transcript. |

Run this command before any stable tag decision:

```bash
devsecops criteria --strict --format json
```

By default it checks `dist/devsecops/production-evidence/v0.12.0`. For the
final stable record, pass the evidence directory that contains the completed
production walkthrough and WSL2 transcript:

```bash
devsecops criteria --strict --evidence-dir dist/devsecops/production-evidence/v1.0.0
```

## Release Candidate Decision

A `v0.12.0` release candidate can be prepared when local tests, package build,
Terraform validation, documentation audit, and security review pass. A stable
`v1.0.0` release must additionally close every blocker above and attach the
final evidence bundle to the release record.
